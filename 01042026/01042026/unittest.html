🎯 Что такое Unit тестирование?

Определение

Проверка корректности работы отдельных функций, методов или классов в изоляции от остальной системы

Цели тестирования

  • Обнаружение ошибок на ранних этапах
  • Упрощение рефакторинга кода
  • Документирование поведения кода
  • Повышение качества разработки
  • Автоматизация проверки функционала
# Пример: тестируем функцию сложения def add(a, b): return a + b # Тест проверяет: # 1. Корректность результата # 2. Обработку граничных случаев # 3. Типы возвращаемых данных

💡 Рекомендация

Используйте pytest для новых проектов. Он проще в освоении, имеет богатую экосистему плагинов и требует меньше кода для написания тестов.

📝 Пример кода для тестирования

Файл: math_ops.py

def add(a, b): return a + b def divide(a, b): if b == 0: raise ValueError("Деление на ноль") return a / b def multiply(a, b): return a * b

Что тестируем?

  • add() - сложение чисел
  • divide() - деление с проверкой на ноль
  • multiply() - умножение

Сценарии тестирования

  • Позитивные числа
  • Отрицательные числа
  • Нулевые значения
  • Исключения (деление на 0)

🧪 Тесты через pytest

Файл: test_math_ops.py

import pytest from math_ops import add, divide def test_add(): assert add(2, 3) == 5 assert add(-1, 1) == 0 assert add(0, 0) == 0 def test_divide(): assert divide(10, 2) == 5.0 assert divide(7, 3) == pytest.approx(2.3333333) def test_divide_by_zero(): with pytest.raises(ValueError, match="Деление на ноль"): divide(5, 0)

✨ Преимущества pytest

  • Обычный assert (не нужны специальные методы)
  • Не нужно создавать классы
  • Автоматическое раскрытие выражений при ошибке
  • Поддержка фикстур и параметризации

🧪 Тесты через unittest

Традиционный подход (встроен в Python)

import unittest from math_ops import add, divide class TestMathOps(unittest.TestCase): def test_add(self): self.assertEqual(add(2, 3), 5) self.assertEqual(add(-1, 1), 0) def test_divide(self): self.assertAlmostEqual(divide(7, 3), 2.3333333, places=5) def test_divide_by_zero(self): with self.assertRaises(ValueError): divide(5, 0) if __name__ == '__main__': unittest.main()

⚠️ Недостатки

  • Многословный синтаксис
  • Обязательное наследование
  • Специальные методы assert

✅ Преимущества

  • Не требует установки
  • Стандарт библиотеки
  • Стабильность API

🚀 Запуск тестов

Команды pytest

# Все тесты в директории pytest # Подробный вывод pytest -v # Конкретный тест pytest -k add # С покрытием pytest --cov=math_ops

Команды unittest

# Все тесты python -m unittest discover # Конкретный файл python -m unittest test_math_ops # С подробностями python -m unittest -v

🎭 Мокирование (Mocking)

Изоляция от внешних зависимостей

from unittest.mock import patch from my_module import fetch_data # Декоратор для мокирования @patch('my_module.requests.get') def test_fetch_data(mock_get): # Настраиваем возвращаемое значение mock_get.return_value.json.return_value = {"status": "ok"} result = fetch_data("https://api.example.com") # Проверяем результат assert result == {"status": "ok"} # Проверяем вызов mock_get.assert_called_once_with("https://api.example.com")

🎯 Когда использовать

  • HTTP-запросы (API)
  • Работа с БД
  • Файловая система
  • Внешние сервисы
  • Сложные вычисления

💡 pytest-mock

Плагин pytest-mock предоставляет удобную фикстуру mocker:

def test_with_mocker(mocker): mocker.patch('os.getcwd', return_value='/tmp')

📦 Покрытие кода (Coverage)

Установка и запуск

# Установка pip install coverage pytest-cov # Запуск с покрытием pytest --cov=math_ops --cov-report=term-missing # HTML отчет pytest --cov=math_ops --cov-report=html

📈 Целевые показатели

  • 80%+ - хороший уровень
  • 90%+ - отличный уровень
  • 100% - не всегда нужен

Пример отчета

Name Stmts Miss Cover Missing ----------------------------------------------- math_ops.py 15 2 87% 8, 12 ----------------------------------------------- TOTAL 15 2 87% Coverage HTML written to dir htmlcov

⚠️ Важно

Покрытие не гарантирует качество тестов! Тесты могут покрывать код, но не проверять корректность.

✅ Лучшие практики

📋 Правила написания

  • Один тест = один сценарий
  • Читаемые имена тестов
  • Тесты независимы
  • Используйте фикстуры

🎯 Что тестировать

  • Happy path (успешные сценарии)
  • Граничные случаи
  • Обработку ошибок
  • Типы входных данных

⚡ Производительность

  • Держите тесты быстрыми
  • Не используйте реальные БД/API
  • Мокайте внешние зависимости
  • Параллельный запуск (pytest-xdist)

🚫 Чего избегать

  • Зависимости между тестами
  • Тестирование тривиальных геттеров
  • Слишком сложных тестов
  • Игнорирования падающих тестов

📄 conftest.py

import pytest # Фикстура доступна во всех тестах @pytest.fixture def sample_data(): return {"key": "value"} # Фикстура с областью session @pytest.fixture(scope="session") def db_connection(): db = create_test_db() yield db db.close()

🎯 Принципы

  • Тесты рядом с кодом
  • Имена файлов: test_*.py
  • Общие фикстуры в conftest.py
